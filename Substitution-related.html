1.1 Substitution-related

1.1.1 Substitution

Substitution in Scala by default is done as follows:

1.1.1.1 Pick the left-most operation.
1.1.1.2 Evaluate its operands.
1.1.1.3 Apply the operator to the operand values.

The process of stepwise simplification of expressions is called reduction.

A function also plays the role of an operator.

Precondition for the following sections: 

scala> def square(x: Double) = x * x
square: (Double)Double

scala> def sumOfSquares(x: Double, y: Double) = square(x) + square(y)
sumOfSquares: (Double,Double)Double

1.1.2 Call-by-value

This is a standard way of doing reduction by the interpreter.

1.1.2.1 The arguments of a function are evaluated (LTR).
1.1.2.2 Function application is replaced by the function's RHS.
1.1.2.3 All formal parameters of the function are replaced by their corresponding actual arguments.
  sumOfSquares(3, 2+2)
→ sumOfSquares(3, 4)
→ square(3) + square(4)
→ 3 * 3 + square(4)
→ 9 + square(4)
→ 9 + 4 * 4
→ 9 + 16
→ 25

1.1.3 Call-by-name

An alternative way of doing reduction

1.1.3.1 Function application is replaced by the function's RHS.
1.1.3.2 The arguments of a function are evaluated (LTR).
1.1.3.3 All formal parameters of the function are replaced by their corresponding actual arguments.
  sumOfSquares(3, 2+2)
→ square(3) + square(2+2)
→ 3 * 3 + square(2+2)
→ 9 + square(2+2)
→ 9 + (2+2) * (2+2)
→ 9 + 4 * (2+2)
→ 9 + 4 * 4
→ 9 + 16
→ 25

Scala uses call-by-value by default, but it switches to call-by-name evaluation if the parameter type is preceded by `=>`.
